# Evaluation Methodology for LLM-Generated Applications

This document outlines the approach used to evaluate applications generated by Large Language Models (LLMs), based on task-specific rubrics and expected system behavior.

---

## Objective

To assess whether LLM-generated applications:

* Meet functional requirements
* Handle state and interaction correctly
* Deliver a consistent user experience

---

## Evaluation Approach

Each task is evaluated using **custom-designed rubrics**, typically consisting of 10–20 criteria across the following dimensions:

---

### 1. Functional Correctness

* Does the application meet core requirements?
* Are all specified rules implemented?

---

### 2. State Management

* Does the system maintain correct internal state?
* Are transitions handled correctly (e.g., timers, thresholds)?

---

### 3. Interaction Logic

* Are user inputs handled correctly?
* Are constraints enforced dynamically?

---

### 4. UI / UX Behavior

* Is the interface responsive and intuitive?
* Are visual indicators aligned with system state?

---

### 5. Edge Case Handling

* Does the system behave correctly under extreme or boundary conditions?
* Are failure conditions handled gracefully?

---

## Scoring System

Each rubric is classified as:

* **Major** → Critical for correctness
* **Minor** → Enhances usability or completeness

Evaluation is based on:

* Pass / Fail per criterion
* Weighted overall performance

---

## Key Insight

LLMs often fail not in isolated logic, but in:

* Multi-condition state transitions
* Implicit constraints
* Time-based behaviors

---

## Role of Human Correction

After evaluation:

* Errors are identified and categorized
* Corrected implementations are created
* These serve as feedback signals for model improvement
