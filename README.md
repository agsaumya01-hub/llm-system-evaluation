# LLM System Evaluation: Failure Design & Diagnostics

This repository showcases my work in evaluating LLM-generated applications by designing complex prompts, defining structured evaluation rubrics, and diagnosing system-level failures.

---

## What This Repository Demonstrates

* Designing **failure-inducing prompts** to test LLM capabilities
* Creating **task-specific evaluation rubrics (10–20 criteria)**
* Analyzing failures in **state management, logic, and interaction flows**
* Providing **human-corrected implementations** for model improvement

---

## Nature of Work

Unlike traditional prompt engineering, this work focuses on:

* Evaluating **end-to-end applications generated by LLMs**
* Testing **multi-condition logic, state transitions, and constraints**
* Identifying **failure patterns in real-world scenarios**

---

## Repository Structure

* `evaluation_methodology.md` → Approach to designing rubrics and evaluating systems
* `case_studies/` → Real examples of prompts, model outputs, and failure analysis

---

## Example Areas of Evaluation

* State synchronization and constraint enforcement
* Time-based logic (e.g., delayed triggers, countdowns)
* UI + interaction consistency
* Edge case handling

---

## Key Insight

LLMs perform well on isolated tasks, but often fail in:

* Coordinated state updates
* Implicit constraints
* Multi-step interaction logic
