## Evaluation Methodology

Each case study follows a consistent evaluation approach:

### 1. Prompt Design
Create **real-world product scenarios** designed to expose model weaknesses.

### 2. Rubric-Based Evaluation
Define **10–20 granular evaluation criteria** across:
- State handling  
- Interaction logic  
- UI behavior  
- Data consistency  

### 3. Failure Analysis
Identify:
- Where the model breaks  
- Why it breaks  
- Patterns across failures  

### 4. Human Correction
Manually fix outputs to:
- Validate expected behavior  
- Provide high-quality training signals  

---

## Common Failure Patterns Observed

Across multiple evaluations, LLMs consistently struggle with:

* **State Synchronization**
  - Keeping multiple components aligned (UI + logic)

* **Multi-Step Dependencies**
  - Cascading updates across variables

* **Constraint Enforcement**
  - Maintaining logical boundaries in systems

* **Dynamic Interactions**
  - Handling real-time updates reliably

* **Aggregation & Ranking Logic**
  - Correctly computing derived metrics
